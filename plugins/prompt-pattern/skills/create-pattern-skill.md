# Create Pattern Skill

사용자가 /patterns에서 특정 패턴을 Skill로 만들기를 원할 때 이 skill이 활성화됩니다.

## 실행 단계

### 1. 패턴 정보 확인

사용자가 선택한 패턴 정보를 확인합니다:
- 대표 프롬프트
- 예시들
- 제안된 Skill 이름

### 2. Skill 이름 확정

사용자에게 Skill 이름을 확인합니다:
- 기본값: 제안된 이름 (예: commit, test, deploy)
- 사용자가 다른 이름을 원하면 변경

### 3. Skill 파일 생성

현재 프로젝트의 `.claude/skills/` 디렉토리에 파일을 생성합니다.

**파일 구조:**

```markdown
---
description: "{패턴 기반 설명}"
---

# /{skillName}

{대표 프롬프트를 기반으로 작성된 명확한 지시사항}

## 실행할 작업

{사용자의 반복 요청을 분석하여 작성된 구체적인 단계}
```

### 4. 생성 완료 안내

사용자에게 알려줄 내용:
- ✅ Skill이 생성되었습니다
- 📍 위치: `.claude/skills/{name}.md`
- 🚀 사용법: `/{name}`
- ✏️ 수정: 파일을 직접 편집하여 개선 가능

### 5. 즉시 테스트 제안

"지금 바로 /{name}을 테스트해볼까요?" 라고 물어봅니다.

## 주의사항

- 기존 Skill과 이름이 충돌하면 다른 이름을 제안
- `.claude/skills/` 디렉토리가 없으면 생성
- 파일 생성 후 반드시 경로와 사용법 안내
